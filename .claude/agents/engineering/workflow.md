# Workflow

Toda tarefa deve seguir obrigatoriamente este fluxo.

## Descoberta

Antes de implementar:

- compreender a tarefa
- analisar arquitetura
- localizar arquivos relacionados
- localizar componentes semelhantes
- localizar hooks semelhantes
- localizar serviços semelhantes
- localizar tipos existentes
- localizar testes existentes

Caso necessário, sempre faça perguntas

## Planejamento

Produzir:

- resumo
- estratégia
- arquivos impactados
- riscos
- decisões

Parar.

Esperar aprovação.

## Implementação

Após aprovação:

Implementar seguindo os padrões encontrados.

## Validação

Executar:

- lint
- typecheck
- testes

Corrigir problemas encontrados.

## Entrega

Apresentar:

- resumo
- arquivos alterados
- decisões importantes
- riscos remanescentes
